# 8917-expert-panel

## 定位

规范型 + Pipeline skill。把「组专家(本地库优先 / 现场生成兜底)→ 并行派遣(反锚定)→ 综合」固化成可重复动作。三模式:讨论 / 评论 / 规划。

**v2 双档执行**:轻型档在对话内手动并行(即时、全程人在环);重型档把"派遣"交给 **Workflow 引擎**(拿到确定性 + fan-in barrier + 规模 + trace)。

选角策略、三模式输出、对抗性 subagent 模板、综合护栏、**Workflow 脚本模板**的完整细节见 [REFERENCE.md](./REFERENCE.md)。

## 双档:走哪个?

| | 轻型档(默认) | 重型档(Workflow) |
|---|---|---|
| 引擎 | 对话内手动并行 Agent 工具 | 生成并调用 Workflow 脚本 |
| 专家数 | ≤5 | 6+ |
| 适合 | 即时小讨论 | 严肃决策 / 要可复现 / 要留痕 / 多轮交锋 |
| 人在环 | 全程 | **前段(确认名单)+ 后段(综合)** |
| fan-in | 手动声明告警 | 代码级 barrier + `missing` 清单 |

默认轻型。命中任一 → 转重型:① 用户说"重大决策 / 可复现 / 留痕";② 专家 ≥6;③ 要多轮交锋;④ 用户说 `--heavy`。**选档前一句话告知用户走哪档。** 环境没有 Workflow 工具 → 自动落轻型档并告知。

## 关键架构铁律(v2)

**Workflow 只当"可靠并行执行器 + barrier",不当综合者。**

重型档拆成三段:**前段**(抽议题 / 选专家 / 确认名单)在对话内 → **Workflow** 只负责"确定性派遣 N 个 + 收齐 + 返回结构化产出 + `missing` 清单" → **后段综合永远回到对话内由我带护栏做**(用户能追问)。人在环两段都在;Workflow 接管"靠谱地把人跑完",不接管判断。

## 三模式

| 模式 | 像什么 | 产出 |
|---|---|---|
| **讨论** | 圆桌会 | 各抒己见 → 共识 + 分歧点 + 各方理由 |
| **评论** | 评审会(对抗) | 挑方案问题 → 分 severity 清单,**强制含一个唱反调** |
| **规划** | 作战会议 | 分领域规划 → 带分工的总计划 |

## 七步流程

1. **抽议题** — 一句话锁定:决定 / 评 / 规划什么 + 成功标准。模糊先问一句。
2. **定模式** — 用户指定就用;没指定按议题推断 + 一句话确认。
3. **选专家(双源)** — 先判来源:**本机有专家库**(会话可见 agency 专家卡)→ 从库选;**无库** → 按议题现场生成专家 persona(角色名 + 单一目标 lens + 立场偏置 + 是否唱反调),用通用 subagent 派遣。轻型 3-5 个,重型 6+,互补优先;**对抗性选角**(评论模式给唱反调喂靶子,REFERENCE §1)。名单里注明来源(库选 / 生成)。
4. **确认名单** — 列「建议召集 X / Y / Z + 理由」,用户增删拍板。**`--auto` / `--heavy` 都不跳过名单确认**(只改执行引擎)。
5. **派遣**(分档):
   - **轻型** → 对话内一条消息并行多个 Agent 工具调用。**派遣前声明"派了 N 个",综合前点"回了 M 个",M<N 在报告顶部告警**(手动 fan-in barrier)。
   - **重型** → 按 REFERENCE §6 生成 Workflow 脚本(`parallel` 派遣 + `agentType` 指定专家 + `schema` 结构化 + `.catch` 兜底),调用 Workflow 工具跑;拿回 `{dispatched, returned, missing, panel}`。
6. **综合**(对话内,带护栏,REFERENCE §3)— **无论哪档,综合都在这里做**。`missing` 非空 → 顶部告警,不用残缺产出伪装完整。
7. **紧凑输出** — 手机可读;各方原始立场在 verdict 前可见,不藏分歧。

## 反锚定铁律(借 council)

- subagent **只拿**议题 + 必要上下文 + 角色,**绝不给完整对话历史**——否则多视角退化成回声。
- 派遣前,我**先写下自己的初判**(立场 + 最强理由 + 最大风险);综合时不简单镜像专家,也不无理由否定。**评论模式**:把我的初判当"靶子"喂给唱反调专家(REFERENCE §4)。

## 边界(不可违反)

- **默认人在环**:第 4 步选完先确认名单;`--auto` / `--heavy` 不跳过名单确认。
- **fan-in 不静默**:任何一档,缺专家必须在报告顶部告警,**严禁用残缺产出伪装成完整综合**。
- **subagent 评审只产判断、不执行**。⚠️ 残留风险:agency 专家卡是 `All tools`,prompt + schema 只是软约束、**不是沙箱**。**不要用本 skill 派攻击性专家(渗透/威胁情报)去真实探测、抓取或改动**;它们只该输出风险描述 + 缓解。
- subagent 零记忆、产出是**初稿 / 多视角原料,非定论**——落地交 8917 角色(Forge/Inspector)或指挥官把关。
- 上下文白名单化:往 subagent 只传议题本身;**敏感路径(`raw/`、`.env`、凭据)默认不带入**。
- 不自动 push / 发布 / 改生产;不替用户做不可逆决策。
- 选不到合适专家(议题太窄/太私域)→ 如实说"库里没贴合的,建议我直接答或换 N 个近似",不硬凑。

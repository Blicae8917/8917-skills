# 8917-write 参考手册

> 配合 `SKILL.md` 使用。四部分：依据块规范、AI 味清单（L3 用）、与 khazix-writer 差异对照、标题模式库。

## 一、依据块格式与核源 SOP（L1 用）

### 格式

正文关键数据 / 结论之后，紧跟一行小字依据块：

```markdown
<small>依据丨来源机构《报告/文章名》(年.月)：一句话摘出关键数字或结论</small>
```

实例（出自《别让 AI 替你思考》）：

```markdown
<small>依据丨微软《The state of global AI diffusion in 2026》(2026.05)：2026 年 Q1
全球 15-64 岁劳动年龄人口生成式 AI 使用率 17.8%，美国 31.3%</small>
```

### 核源 SOP

1. AI 检索到的出处只是**线索**——必须逐个验证真实存在（搜标题、开链接、对数字），验证不了的不用
2. 一个关键结论至少一个独立出处；出处之间互相打架时，正文如实写「两组数据有出入」
3. 转引（A 报道 B 的数据）标到最源头；找不到源头就标「转引自 A，原始出处未核实」并降级表述
4. 自有数据（Agent 运行数据、项目量化结果）标注口径与时间点，如「2026-07 快照」
5. 事实 / 推断 / 未知三分话术：
   - 事实：「数据是…」＋依据块
   - 推断：「我的判断是…」「我猜…」（不加依据块，不装成事实）
   - 未知：「这个我没查到 / 我不知道」

## 二、AI 味清单（L3 扫描用，摘自 humanizer-zh，MIT，源自维基百科 AI 写作特征）

逐模式扫描成稿。命中即改，改法参考各条「改成」。

**⚠ 金句豁免条款**：humanizer 原版要求「删除听起来像可引用的语句」。本 skill 的**原则化金句**（每篇 ≥1，可进 wiki / 上封面）豁免此条——金句是叶氏特性。豁免仅限真正的原则金句，其余顺口溜式「响亮空话」照删。

### 内容模式

| # | 模式 | 雷词 / 特征 | 改成 |
|---|---|---|---|
| 1 | 夸大意义 | 标志着、见证了、凸显了其重要性、关键转折点、不断演变的格局、深深植根于 | 直接陈述事实本身 |
| 2 | 堆砌知名度 | 「被 X、Y、Z 引用」「拥有活跃的社交媒体存在」 | 给一个具体场景，删名单 |
| 3 | -ing 式肤浅收尾 | 句尾挂「反映了…」「彰显了…」「为…做出贡献」 | 删掉或换成具体归因（谁说的、为什么） |
| 4 | 宣传腔 | 充满活力的、丰富的、令人叹为观止、必看、开创性的 | 中性名词＋具体事实 |
| 5 | 模糊归因 | 专家认为、观察者指出、行业报告显示（无具体来源） | 具体到机构＋年份，或删（与依据块规范同构） |
| 6 | 公式化「挑战与展望」 | 尽管存在这些挑战…未来可期 | 写具体的事件与具体的应对 |

### 语言模式

| # | 模式 | 雷词 / 特征 | 改成 |
|---|---|---|---|
| 7 | 高频 AI 词 | 此外、至关重要、深入探讨、赋能、格局、织锦、宝贵的、彰显 | 换白话或删 |
| 8 | 回避「是」 | 作为 / 充当 / 拥有 / 设有 | 直接用「是」「有」 |
| 9 | 否定式排比 | 「不仅仅是…而是…」「这不只是…」 | 只留正面陈述 |
| 10 | 三段式强迫症 | 凡列举必三项 | 两项优于三项；该几项就几项 |
| 11 | 同义词循环 | 主人公→主要角色→中心人物→英雄 | 就用同一个词 |
| 12 | 虚假范围 | 「从 X 到 Y」但 X、Y 不在一个尺度上 | 老实列举 |

### 卡兹克指纹检测（本 skill 附加）

以下出现即改——这些是参考作者的身份指纹，不是通用技巧：

- 口癖：太特么赤鸡了、不是哥们、我寻思了一下我没寻思明白、有个屁的 X、这尼玛
- 情绪标点人设：「。。。」「？？？」「= =」作为风格元素反复出现
- 固定尾巴：点赞在看转发三连＋星标请求＋「磨平信息差」使命句
- 高密度单句断行（每两三段一次）——叶氏版全文 3 处左右

## 三、与 khazix-writer 的差异对照（速查）

| 维度 | khazix-writer | 8917-write |
|---|---|---|
| 身份 | 卡兹克本人 | 叶澄风（卡兹克降为参考作者） |
| 第一层质检 | 禁用词 / 禁用标点扫描 | **证据核验**（依据块覆盖率、出处真实性、三分标注） |
| 小标题 | 原则禁止 | 叙事文不加；方法论 / 复盘文用编号节＋表格＋代码块 |
| 标点禁令 | 冒号 / 破折号 / 引号硬禁 | 降级为 AI 味提醒（清单 #13） |
| 幻觉策略 | 容忍幻觉（创意前提），靠外部模型事后核查 | 证据链内置于写作流程，写时即核 |
| 金句 | 无显式要求 | 每篇 ≥1 原则化金句（humanizer 豁免） |
| 交付物 | 创意案例「卧槽感」 | 每篇 ≥1 可直接拿走的 prompt / 清单 / 模板 |
| 选题判据 | HKR | HKR ＋ 资产判据（沉淀什么可复用资产） |
| 情绪表达 | 高密度口癖＋情绪标点 | 克制版口语，无情绪标点人设 |
| 尾部 | 固定三连＋署名邮箱 | 无固定尾巴；回扣 / 行动 / 留白收尾 |

继承不变的：现场感开场、先判断后展开、知识随手掏、对立面理解、谦逊铺垫、逐一展示＋升番、四层自检架构、「AI 初稿 vs 人改稿」迭代法。

## 四、标题模式库（第四步用）

成文后出 5-8 个候选，按 HKR 排序。四个基础模式（可组合）：

| 模式 | 结构 | 例（卡兹克 corpus 实例） |
|---|---|---|
| 数字量化 | 「N 个 / 从 X 到 Y」 | 《想玩 Loop Engineering，可以先从这 6 个 Hook 玩法开始》《我把 15 个 AI Agent 的记忆架构，从 25KB 压到了 4KB》 |
| 悬念缺口 | 结果不说透 | 《今年高考，我让 12 个顶级 AI 一起考了语文和数学，结果有点意外》 |
| 身份反差 | 反常识的主客关系 | 《给机器人打工了一天》《2026 年的毕业生们，正在花钱向 AI 证明自己是人类》 |
| 第一人称动作 | 「我把 / 我让 / 我用…做了…」 | 《我把斯坦福最火的一门课，做成了 Prompt 来帮我设计人生》 |

叶氏加权规则：

- 标题里的数字必须是正文里核实过的真数字（L1 纪律延伸到标题）
- 不用「震惊 / 炸裂 / 变天」类透支信任的词
- 对外标题不出现内部代号（8917 / Agent 真名）
- 候选中至少一个「资产型标题」——突出读者能拿走什么（「一套 / 一份 / 拿走即用」）

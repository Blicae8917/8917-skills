# 内容抓取方法

## 目标
将不同来源的内容提取为可读正文，优先速度、稳定性和低 token 成本。

---

## 1. X / Twitter
优先使用：
- `xreach tweet <URL> --json`

用于：
- 获取推文正文
- 获取长文线索
- 获取上下文信息

---

## 2. 通用网页 / 博客 / 新闻 / 公众号正文
优先使用：
- `https://r.jina.ai/<URL>`

示例：
```bash
curl -s "https://r.jina.ai/<URL>" -H "Accept: text/markdown"
```

目标：
- 快速拿到 Markdown 正文
- 避免浏览器自动化

---

## 3. 微信公众号兜底
如果 Jina 抓取失败，可尝试：
- `web_fetch`

---

## 4. 视频平台
对于抖音 / B站 / YouTube 等视频内容：
- 使用 `8917-local-whisper` 提取音频和字幕
- 输出 transcript 供后续分析或归档

---

## 5. 方法选择原则
1. 优先轻量方法，不要默认浏览器自动化
2. 优先结构化正文，不要抓整页噪音
3. 失败时明确说明，并给出下一步兜底方案
4. 抓取层只负责“拿内容”，不负责分析和归档

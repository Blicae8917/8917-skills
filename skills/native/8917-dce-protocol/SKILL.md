# 8917-dce-protocol

## 定位

这是一个 **规范型 / Inversion + Pipeline** skill。

它负责把 DCE（Discuss → Confirm → Execute）变成 agent 可直接执行的行为协议。

它只保留：
- 触发条件
- 执行顺序
- 确认边界
- 模板资产

完整理论、设计动机、组织治理语境，请见：
- `8917-organization-mastery`

---

## 何时触发 DCE

遇到以下任务，默认触发 DCE：
1. 配置修改
2. 外部通信（邮件、社交媒体、公开发布）
3. 数据删除、权限变更
4. 安全相关操作
5. 不可逆操作（发布、部署、迁移、转移）
6. 高资源消耗任务
7. 首次任务 / 非 SOP 任务

---

## 核心流程

### Step 1：Discuss
先给出 2~3 个方案，并说明：
- 各自优缺点
- 风险点
- 推荐路径

必要时补充关键问题，但不要没完没了追问。

### Step 2：Confirm
只有收到明确确认，才允许进入执行。

#### 明确确认示例
- 执行
- 确认
- 批准
- Proceed
- `1`（在 8917 沟通语境中代表同意 / 继续）

#### 不算确认
- 好的
- OK
- 可以（若上下文不明确）
- 任何模糊回应

### Step 3：Execute
确认后再执行，并在必要时：
- 留下结果摘要
- 记录复盘线索
- 交给收尾链路继续处理

---

## 边界

本 skill 不负责：
- 完整组织管理理论说明
- 长篇方法论教育
- 记忆系统的全部架构解释

这些属于：
- `8917-organization-mastery`

本 skill 只负责把 DCE 变成可执行的 agent 行为协议。

---

## 一句话原则

**先讨论，后确认，再执行；DCE 在 `8917-skills` 里是可执行协议，不是理论全文。**
